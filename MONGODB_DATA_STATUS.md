## ✅ API Endpoints Using MongoDB

### `/api/products` (GET)
**File:** `/app/api/products/route.ts`

```typescript
// Connects to MongoDB
await dbConnect();

// Queries Product model
const products = await Product.find(query)
  .sort(sortCriteria)
  .skip(skip)
  .limit(limit)
  .populate('seller', 'name email')
  .lean();
```

**Supports:**
- ✅ Pagination (`page`, `limit`)
- ✅ Search (`search` - regex on title/description/tags)
- ✅ Category filtering (`category`)
- ✅ Price range (`minPrice`, `maxPrice`)
- ✅ Featured products (`featured=true`)
- ✅ Seller products (`sellerId`)
- ✅ Sorting (by price, rating, date, name)

**Returns:**
```json
{
  "success": true,
  "products": [...],
  "pagination": {
    "page": 1,
    "limit": 12,
    "total": 100,
    "pages": 9
  }
}
```

## 🔧 How to Seed Database with Products

### Option 1: Use Migration Endpoint

```bash
# Check current product count
curl http://localhost:3000/api/admin/migrate-products

# Seed 50 products (requires NEXTAUTH_SECRET from .env)
curl -X POST http://localhost:3000/api/admin/migrate-products \
  -H "Content-Type: application/json" \
  -d '{"count": 50, "secret": "YOUR_NEXTAUTH_SECRET"}'
```

### Option 2: Use the Check Script

```bash
# Run the automated check and seed script
./scripts/check-products.sh
```

This script will:
1. Check if products exist in MongoDB
2. Auto-seed if database is empty
3. Show sample products
4. Provide test commands

### Option 3: Manual Seeding via API

Start the dev server and visit:
```
http://localhost:3000/api/admin/migrate-products
```

Then POST with authentication.
